**What changed and why.** I integrated a four-lens adversarial review (security, scalability, MVP-pragmatism, completeness) into the plan. The architecture's spine is sound and is kept verbatim; the fix was separating the *north-star target* from the *day-one starting surface*. I independently verified the panel's two highest-stakes 2026 claims, and both held: (1) the `snow` Noise crate still has **no fallback modifier**, so the draft's flagship "Noise Pipes / IK-with-XXfallback" was **unbuildable**; (2) Azure Trusted Signing's **2026-03-26 CA rotation reintroduced SmartScreen warnings** until reputation accrues. Both reshaped the plan.

**The five load-bearing corrections.**
1. **Buildable crypto (ADR-0003):** replaced XXfallback with `Noise_XX` first contact → cached `Noise_IK` reconnect → clean SAS re-pair on key change (what `snow` can actually build), never hand-rolling the handshake.
2. **Transport reversal (ADR-0002):** the MVP/v1 native data plane is now **quinn/QUIC** (stable, gives encryption + congestion control + connection migration), with **webrtc-rs deferred to Phase 3** for the browser only — isolating its documented 2026 instability (v0.17 memory leak, v0.20 still an RC) to the one phase that needs a browser.
3. **Ruthlessly thin Phase 1:** a single host binary + single desktop client in **one crate**, LAN-only, manual-IP/mDNS, **hand-written wire structs (no proto/buf)**, **winit+wgpu (no Tauri/TS)**, Noise-encrypted, SAS pairing, with an explicit "cannot control UAC/lock screen yet" banner. Concrete code first; **traits extracted from working code, not written ahead of it.**
4. **Defer ~60–70% of the tree as ADR-reserved slots (not deletions):** OPAQUE accounts, SFrame, the api-gateway tier, the full TS/Turborepo toolchain, the 7-crate `core/` split, `cargo-hakari`, the bespoke dep-direction gate, k8s/terraform/ansible, four FFI surfaces — each lands in the phase that forces it, tracked in a deferred-slot ledger.
5. **Closed real gaps as new sections:** distribution/store gatekeepers (iOS *host* is infeasible; desktop is the resolved day-one dogfood — Open Q#1), a concrete test strategy (fuzzing from Phase 0, NAT matrix, relay-abuse), security operations + a device-loss recovery design, client diagnostics, legal-beyond-codecs (crypto export/EAR, NVENC redistribution, GDPR-as-relay-operator), and a platform/capability matrix. The session-0 secure-desktop helper is now its own subcrate with an IPC trust-boundary ADR (0008), deferred to Phase 2.
